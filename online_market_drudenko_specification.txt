Техническое задание:
Заголовок страницы: Страница доступа
На странице необходимо отобразить форму для входа на веб версию сайта
Отображаться должны следующие данные:
-	Email
-	Password

Заголовок страницы: Пользователи
На странице необходимо отобразить список пользователей.
Страница доступна только для пользователя с привилегией Administrator.
Отображаться должны следующие данные:
-	ФИО 
-	Email 
-	Role
Записи должны быть отсортированы по email в алфавитном порядке. 
Если записей более 10, должен осуществляться постраничный вывод. 
Для каждой записи должна быть возможность пометки на удаление (чекбокс). 
Под списком-таблиц должна быть кнопка “Удалить”, по нажатию на которую все 
помеченные записи должны быть удалены. Один из администраторов с ролью 
“администратор” должен быть недоступен для удаления (и ему должна быть 
заблокирована возможность “уменьшить” привилегии), чтобы не было возможности 
удалить всех пользователей. 
Для каждой записи должна быть возможность изменения пароля, который 
генерируется автоматически произвольно и отправляется на email пользователя. 
Для каждой записи должна быть возможность изменения привилегий. 

Заголовок страницы: Добавление пользователи
На странице необходимо отобразить форму для добавления пользователя. 
Страница доступна только для пользователя с привилегией Administrator.
Заполняться должны следующие данные:
-	Фамилия(40 символов, только латинские буквы)
-	Имя(20 символов, только латинские буквы)
-	Отчество(40 символов, только латинские буквы)
-	Email(50 символов, стандартный шаблон)
-	Role(Administrator, Sale User, Customer User, Secure API User)

Заголовок страницы: Отзывы
На странице должен быть отображен список отзывов. 
Отображаться должны следующие данные: 
-	ФИО
-	Отзыв 
-	Дата добавления 
-	Статус “Показывать” (чекбокс) 
Если записей более 10, должен осуществляться постраничный вывод. 
Также на данной странице должна быть возможность удалить отзыв.

Заголовок страницы: Articles (/sale/articles)
На странице необходимо отобразить список новостей.
Страница доступна только для пользователя с привилегией Sale User.
Отображаться должны следующие данные:
-	Заголовок
-	Текст
-	Дата создания
Новости должны быть отсортированы по дате создания. 
Под каждой новостью должна быть кнопка “Открыть комментарии”, по нажатию на 
которую происходит переход на страницу комментариев для выбранной новости.
Под каждой новостью должна быть кнопка “Изменить”, по нажатию на которую 
происходит переход на страницу изменения новости.
Если новостей более 10, должен осуществляться постраничный вывод. 
Для каждой новости должна быть возможность пометки на удаление (checkbox). 
Под списком новостей должна быть кнопка “Удалить”, по нажатию на которую все 
помеченные новости должны быть удалены.
Под списком новостей должна быть кнопка “Создать”, по нажатию на которую 
происходит переход на страницу создания новостей.

На странице необходимо отобразить новость с id = {id} и список комментариев для 
этой новости. Страница доступна только для пользователя с привилегией Sale User.
Отображаться должны следующие данные:

-	Текст новости
-	Дата создания новости

Комментарии должны быть отсортированы по дате создания. 
Если комментариев более 10, должен осуществляться постраничный вывод. 
Для каждого комментария должна быть возможность пометки на удаление (checkbox). 
Под списком комментариев должна быть кнопка “Удалить”, по нажатию на которую все 
помеченные комментарии должны быть удалены.

Заголовок страницы: Items (/sale/items)
На странице необходимо отобразить список товаров.
Страница доступна только для пользователя с привилегией Sale User.
Отображаться должны следующие данные:
-	Имя
-	Описание
-	Уникальный номер
-	Цена
Товары должны быть отсортированы по имени в алфавитном порядке. 
Под каждым товаром должна быть кнопка “Копировать”, по нажатию на которую 
происходит переход на страницу создания товаров, но с заранее заполненными 
полями, как у выбранного нами товара.
Если товаров более 10, должен осуществляться постраничный вывод. 
Для каждого товара должна быть возможность пометки на удаление (checkbox). 
Под списком товаров должна быть кнопка “Удалить”, по нажатию на которую все 
помеченные товары должны быть удалены.
Под списком товаров должна быть кнопка “Создать”, по нажатию на которую 
происходит переход на страницу создания товаров.
Под списком товаров должна быть кнопка “Загрузить”, по нажатию на которую 
происходит переход на страницу загрузки товаров из файла xml.

Заголовок страницы: Create item (/sale/articles/create)
На странице необходимо отобразить форму для создания товаров. 
Страница доступна только для пользователя с привилегией Sale User.
Заполняться должны следующие данные:
-	Имя (от 1 до 30 символов (VARCHAR(30)), только латинские буквы, пробелы, 
дефисы)
-	Описание (от 1 до 100 символов (VARCHAR(100)), только латинские буквы, 
цифры, пробелы, дефисы)
-	Цена (от 1 до 15 цифр до запятой, от 1 до 5 цифр после запятой (если она есть) 
(DECIMAL(15,5)))
Если переход на страницу произошёл через кнопку “Копировать”, поля заранее 
заполнены и их можно изменить на своё усмотрение.

Заголовок страницы: Upload items (/sale/items/upload)
На странице необходимо отобразить форму для загрузки файла со списком товаров в 
формате xml. Страница доступна только для пользователя с привилегией Sale User.

Заголовок страницы: Orders (/sale/orders)
На странице необходимо отобразить список заказов всех пользователей.
Страница доступна только для пользователя с привилегией Sale User.
Отображаться должны следующие данные:
-	Имя покупателя
-	Адрес покупателя
-	Телефон покупателя
-	Email покупателя
-	Имя товара
-	Количество товара
-	Суммарная цена
-	Дата создания заказа
-	Статус заказа
Заказы должны быть отсортированы по дате создания. 
Если заказов более 10, должен осуществляться постраничный вывод. 
У каждого заказа есть выпадающий список на изменение статуса и соответствующая 
кнопка. Текущий статус заказа должен быть заблокирован в этом списке.
